# ☕ CoffeeLine API
Це серверна частина проєкту **CoffeeLine**, написана на **Spring Boot**. 
Даний RESTful API забезпечує всю необхідну бізнес-логіку для керування замовленнями, 
кавовими продуктами, їх категоріями, користувачами та автентифікацією.

## Можливості
- **Автентифікація:** Реєстрація (`Sign Up`) та вхід (`Sign In`) за допомогою JWT.
- **Авторизація:** API захищено за допомогою Spring Security з різними ролями (`USER`, `CASHIER`, `ADMIN`).
- **Управління Категоріями:** Повний CRUD для категорій кави (тільки для `ADMIN`).
- **Управління Кавою:** Повний CRUD для кавових продуктів (тільки для `ADMIN`).
- **Управління Замовленнями:** Створення замовлень, перегляд замовлень за статусом або користувачем, зміна статусу замовлень.
- **Управління Користувачами:** Перегляд та керування ролями користувачів (тільки для `ADMIN`).

## Технологічний стек
- **Версія Java:** 21
- **Фреймворк:** Spring Boot
- **Архітектура:** Багатошарова
- **Безпека:** Spring Security (з автентифікацією JWT)
- **Управління даними:** Spring Data JPA
- **База даних:** PostgreSQL
- **Документація:** Springdoc OpenAPI 3 (Авто-генерація Swagger UI)
- **Build-система:** Gradle

## Перед початком роботи
1. **Клонуйте репозиторій:**
   
   `git clone https://github.com/Lisovskiy14/CoffeeLine-Backend.git`
3. **Налаштуйте базу даних:**
   - Створіть нову базу даних PostgreSQL.
   - Перевірте конфігурацію в `application.yml`. Якщо необхідно, налаштуйте конфігурацію підключення до БД відповідно до ваших потреб.
4. **Зберіть та запустіть проєкт.**

## Документація API (OpenAPI)
Після запуску сервера, інтерактивна Swagger UI доступна за адресою: 

http://localhost:8080/swagger-ui.html

JSON-специфікація OpenAPI доступна за адресою: 

http://localhost:8080/v3/api-docs

```
src/main/java/com/example/CoffeeLine/
├── common/          # Спільні класи (Enums)

├── domain/          # Сутності (База Даних)

├── security/        # Безпека (JWT, Фільтри)
├── service/         # Бізнес логіка
├── util/            # Корисні утиліти
└── web/             # Контролери (API), мапери
```
